01 — Context, User & Problem 01 — 背景、用户与问题

The prompt is the highest-leverage input in vibe coding. Prompt 是 Vibe Coding 中影响力最大的单一输入。

Vague prompts force the AI to infer intent, resolve ambiguity, and backtrack — wasting tokens and developer time on every single run. 模糊 prompt 迫使 AI 推断意图、消解歧义并反复修正，每次运行都在白白消耗 token 和开发者时间。

Who 目标用户

Vibe Coding Developers Vibe Coding 开发者

Software developers using AI coding tools — Cursor, GitHub Copilot, Claude — who describe what they want in plain language and let AI write the code. 使用 Cursor、GitHub Copilot、Claude 等 AI 编程工具的开发者，习惯用自然语言描述需求，让 AI 直接生成代码。

Workflow Improved 优化的工作流

The Prompt Moment 输入 Prompt 的关键时刻

The exact moment a developer types a request into a coding AI. This single input determines whether AI writes correct code on the first try — or requires multiple costly rounds. 开发者向 AI 编程工具输入请求的那一刻。这个输入决定了 AI 能否一次生成正确代码，还是需要多轮昂贵修正。

Why GenAI Is Necessary 为什么必须用 GenAI

Rules Can't Understand Intent 规则引擎无法理解意图

A rule engine can strip filler words, but cannot determine which context is semantically relevant, what constraints a framework choice implies, or whether a sentence is redundant given the overall goal. 规则引擎能删除填充词，但无法判断哪些上下文语义相关、框架选择隐含哪些约束，或某句话在整体目标下是否冗余。

Mentor ConversationMentor 对话机制

Before optimization, a 5-10 turn chat collects language, framework, I/O format, constraints, and edge cases — one question at a time. The pipeline only runs after the developer confirms.优化前，系统通过 5-10 轮对话逐一收集语言、框架、I/O 格式、约束条件和边界情况，开发者确认后流水线才启动。

Human-in-the-Loop人在回路设计

TokenLess surfaces quality scores and refuses to output prompts that fail the quality gate (overall < 6.0). Low scores and edge cases are handed back to the developer explicitly.TokenLess 展示质量评分，拒绝输出未通过门槛（综合分 < 6.0）的 prompt。低分和边界情况明确交还给开发者判断。

## Role
You are a senior React/TypeScript developer. Your expertise: real-time data dashboards and financial UI components.

## Task
Build a real-time stock price dashboard with Recharts visualization and price-movement alerts.

## Context
- Framework: React 18 + TypeScript
- Charting: Recharts
- Data source: WebSocket connection to a stock price API (endpoint TBD)
- Alert threshold: ±5% price movement triggers an in-app notification

## Constraints
- No user authentication required
- Use functional components and React hooks only
- Handle WebSocket reconnection gracefully

## Output Format
Return the answer using this JSON-compatible structure:

```json
{
  "files_to_create_or_modify": [],
  "implementation_steps": [],
  "acceptance_criteria": [],
  "tests": []
}
```

## Reminder
Implement the WebSocket connection and alert logic first; the chart component is secondary.
